# Feature: Internationalization (FR/EN)

## Overview
French is the primary language. English is important because of the proximity to
La Cité des Congrès (international conferences). English should be genuinely good,
not Google Translate quality.

## Strategy: next-intl

### Routing
```
/           → redirects to /fr (or serves FR based on Accept-Language)
/fr/...     → French pages
/en/...     → English pages
```

### Middleware
```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware';

export default createMiddleware({
  locales: ['fr', 'en'],
  defaultLocale: 'fr',
  localeDetection: true, // Use Accept-Language header for first visit
});
```

### Message Files
```
src/i18n/messages/
├── fr.json     # French (source of truth)
└── en.json     # English (manually translated, not auto)
```

### What Gets Translated
**Static UI elements** (via next-intl messages):
- Navigation labels
- Button text
- Form labels and placeholders
- Error messages
- Footer content
- Page headings and static copy

**CMS content** (via Payload localization):
- Daily menu: FR only (French menu terminology is expected, even for English speakers)
- Wine card: FR only (wine names are universal, descriptions optional EN)
- About page: FR + EN
- Group meals: FR + EN
- Site settings (hours, etc.): universal, no translation needed

### Payload Localization Config
```typescript
// payload.config.ts
{
  localization: {
    locales: [
      { label: 'Français', code: 'fr' },
      { label: 'English', code: 'en' },
    ],
    defaultLocale: 'fr',
    fallback: true, // Fall back to FR if EN not provided
  }
}
```

### Language Switcher
- Simple toggle in header: "FR | EN"
- Preserves current page when switching
- Uses next-intl's `useRouter` and `usePathname`

### SEO
- `<link rel="alternate" hreflang="fr" href="...">` on all pages
- `<link rel="alternate" hreflang="en" href="...">` on all pages
- `<html lang="fr">` or `<html lang="en">` set correctly

### Translation Quality
- All EN translations written manually (not machine-translated)
- Tone: warm, slightly informal — same energy as the French
- Menu items stay in French even on EN pages (with optional brief EN description)
- "Fait maison" → keep in French or use "Homemade" — decide based on context
